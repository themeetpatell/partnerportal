Design Section 7 — Public API, Webhooks & Phase Breakdown

REST API for white-label partners + phased delivery roadmap

Public REST API — apps/api

Authentication
API Key in header: X-API-Key: sk_live_...
Keys scoped per partner + per tenant
Scopes: leads:write leads:read
          services:write commissions:read
Rate limiting: 1,000 req/hour per key
Key management UI in partner portal
Base URL
api.finanshels.com/v1/

All responses:
{ data, meta, error }

Errors:
{ code, message, details }
Endpoint Method Description
/leads POST Submit a new lead
/leads/{id} GET Get lead status & details
/leads GET List partner's leads (paginated)
/service-requests POST Submit a service request
/service-requests/{id} GET Get service request status
/commissions GET List commissions earned
/commissions/summary GET Total earned, pending, paid
/services GET Service catalog
/webhooks POST Register a webhook endpoint
/webhooks/{id} DELETE Remove a webhook

Webhook Events Delivered to Partners

Lead Events
lead.submitted
lead.in_review
lead.qualified
lead.converted
lead.rejected
Service Events
service.received

service.completed
service.doc_requested
Finance Events
commission.earned
commission.paid
invoice.issued
invoice.paid
invoice.overdue
Delivery: HTTPS POST with X-Finanshels-Signature (HMAC-SHA256). Retry: 3 attempts with exponential backoff. Delivery logs visible in partner portal.
