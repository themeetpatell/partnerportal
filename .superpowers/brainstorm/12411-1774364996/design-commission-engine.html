Design Section 5 — Commission Engine

Shared package packages/commission-engine — pure TypeScript, testable, reused by all apps

Three Commission Models

Model 1 — Flat %
Fixed percentage of the service fee on every converted lead or completed service request.
config: { pct: 10 }

commission =
  service_fee × (pct / 100)
Example: AED 5,000 fee → AED 500
Model 2 — Tiered
Commission % increases as the partner converts more leads in a rolling period (monthly/quarterly).
config: {
 tiers: [
  { min:0, max:5, pct:8 },
  { min:6, max:10, pct:12 },
  { min:11, max:null, pct:15 }
 ], period: "monthly"
}
8th lead this month → 12% rate
Model 3 — Milestone
One-time bonus rewards when partner hits cumulative conversion targets.
config: {
 milestones: [
  { target:10, reward:500 },
  { target:25, reward:1500 },
  { target:50, reward:5000 }
 ], currency: "AED"
}
25th conversion → AED 1,500 bonus

When Commissions Are Triggered

Referral Partner Trigger
Event: Lead status → Converted
Source: Sales team marks lead converted in admin portal OR Zoho Deal stage = "Closed Won" webhook
Engine: calculates amount based on partner's model
Record: commission row created with status pending
Notification: partner emailed + in-app alert
Channel Partner Trigger
Event: Service request status → Completed
Source: Ops team marks service complete in admin portal
Engine: calculates commission on service fee
Record: commission row → added to next invoice period
Invoice: auto-generated on billing cycle (weekly/monthly)
