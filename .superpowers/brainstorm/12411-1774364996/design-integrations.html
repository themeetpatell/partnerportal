Design Section 6 — Integrations

Zoho CRM · Zoho WorkDrive · Stripe · SendGrid — all wrapped in packages/zoho, packages/notifications

Zoho CRM — Bidirectional Lead Sync

Portal → Zoho (Push)
On lead submission:
→ Create Zoho Lead via API
→ Store zoho_lead_id on our lead record

On lead qualified (sales):
→ Convert Zoho Lead → Zoho Deal
→ Store zoho_deal_id
→ Update deal stage to "Proposal"

On lead converted:
→ Update Zoho Deal stage to "Closed Won"
Zoho → Portal (Webhook)
Zoho workflow triggers webhook when:
→ Deal stage = "Closed Won"
  → auto-mark lead Converted
  → trigger commission calculation

→ Deal stage = "Closed Lost"
  → mark lead Rejected
  → notify partner

→ Lead owner changed
  → update assigned_to in portal
Auth: Zoho OAuth 2.0 (server-side token refresh stored encrypted in DB per tenant) · Rate limit: 5,000 API calls/day (free) → handled with queuing

Zoho WorkDrive — Document Management

Folder Structure
Partner Portal/
 ├── Partners/
 │   └── {partner_id}/
 │       ├── KYC/
 │       └── Agreements/
 ├── Leads/
 │   └── {lead_id}/
 └── ServiceRequests/
     └── {request_id}/
Upload Flow
1. Partner selects file in portal
2. Server requests upload URL from WorkDrive API
3. File streamed to WorkDrive
4. WorkDrive returns file ID + URL
5. We store reference in documents table
6. Partner sees uploaded doc with preview link
Access Control
Partners: upload only to own folder
Admin/Ops: read all folders
Download: signed URL (expires 1hr)
Delete: admin only

Auth: Zoho OAuth 2.0
(same token as CRM, different scope)

Stripe — Payouts & Invoicing

Referral Partner Payouts
Partners onboard with Stripe Connect (Express account)
→ Finance approves commission batch
→ stripe.transfers.create() to partner's connected account
→ Stripe webhook transfer.paid → mark commission Paid
→ Partner gets email notification

Statement: downloadable PDF per period
Channel Partner Invoicing
→ Auto-generate Stripe Invoice weekly/monthly
→ Email invoice to channel partner via SendGrid
→ Partner pays via Stripe-hosted invoice page OR
   embedded Stripe checkout in portal
→ Webhook invoice.paid → mark invoice Paid
→ Webhook invoice.overdue → SendGrid reminder

Tax: VAT line items via Stripe Tax (UAE 5%)

SendGrid — Notification Emails

Onboarding
✉ Welcome + login link (on approval)
✉ Registration received (pending)
✉ Rejection with reason
✉ Additional docs requested
Lead & Service
✉ Lead received confirmation
✉ Lead status change
✉ Lead converted (commission earned!)
✉ Service request status update
✉ Service completed
Finance
✉ Commission approved
✉ Payout processed
✉ Invoice issued (channel)
✉ Payment received
✉ Invoice overdue reminder (D+7, D+14)
All templates stored in SendGrid Dynamic Templates. Variables injected server-side. Unsubscribe handled by SendGrid preference centre.
